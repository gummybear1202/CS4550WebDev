User First Design

    (5 user stories and description of the process)

  1. “As a returning user, I should be able to log in to my account”

    2. Description:

    • Alice visits the blogging app
    • She inputs her username and password
    • She clicks on the login button
    • she is logged in successfully
    • she is brought to her account home page

  2. “As an user, I should be able to search other people by their account names”

    3. Description:

    • Carol visits the blogging app
    • She logs in with her username and password
    • She is brought to her account home page
    • She finds the search box and selects ‘users’ from the types of search
    • She inputs the username, Carol, which she wishes to find
    • She is brought to the accounts page with relevant names
    • She finds the right user she wants to find

  3. “As an user, I should be able to post comments”

    4. Description:

    • David visits the blogging app
    • He logs in with his username and password
    • He is brought to his account home page
    • He sees a list of previous comments
    • He enters the text of his comments in the textfield
    • He clicks on ‘Post’
    • His new comment is now displaying on his home page at the top

  4. “As an user, I should be able to reset my password when I forget and cannot login successfully”

    5. Description:

    • Eddy visits the blogging app
    • Eddy enters the username and password
    • Eddy sees a message that whether his username or his password is wrong
    • Eddy knows his username is correct, but cannot remember his password
    • Eddy clicks on ‘Forget Password’
    • Eddy is brought to a new page
    • Eddy enters his email address and clicks ‘send’
    • Eddy receives an email and clicks on the link to reset his password
    • Eddy is brought to another page to enter a new password
    • Eddy enters his new passwords in both textfields (second to confirm)
    • Eddy clicks on ‘submit’
    • Eddy is now brought to the blogging app login page
    • Eddy logs in with his username and new password

  5. “As an user, I should be able to use # as hashtags and @ to mention other users when commenting”

    6. Description:

    • Frank visits the blogging app
    • Frank logs in with his username and password
    • Frank writes a comment with some hashtags and @Tom, who is another user he wants to mention
    • Frank clicks ‘Post’
    • Frank sees the new comment up top on his home page with the hashtag and mentions colored differently
    • When Tom logs in, he receives a notification that he has been mentioned in Frank’s comment

Data First Design

as for update, it should appear similar to the show page with message
for delete, it should appear similar to the index page with message

Users

Data

  • email
  • account name (apear to others)
  • password
  • list of comments posted
  • list of users (following)
  • list of users (follower)

constraints: each user in the list should be unique (no repetition)

Application Pages

new/create :

  • have 3 editable textfields: email, account name, password
  • have a create button at the bottom to create the user

    • show :

  • has the user as a card
  • list of comments that are posted by the user
  • has a button for creating a new comment

    • index :

  • list of existing users
  • has columns for each data fields, but only references for each list

    • Comments

    Data

    • text
    • list of special references (hashtag and mentions)
    • owner
    • date

    constraints: a comment can have multiple references (#dog #cat #….etc.); however, the list should not contain identical data (#dog #dog shouldn’t be stored twice)

    Application Pages

    new/create :

  • have an editable textfield to write the comment and a submit button

    • show :

  • display the comment in text with time, date, and the owner

    • index :

  • have each comment separate as a card

    • Special References

    Data

    • leading symbol ('#' or '@')
    • identification (a hashtag or a mention)

    constraints: other symbols should be ignored (!, @, $, etc.) when detecting the leading symbol

    Application Pages

    new/create :

  • ideally should be embedded in the comment’s form

    • show :

  • when a comment includes a special reference, show it by colorcoding the identification differently

    • index :

  • columns include: 1) owner 2) identification 3) content that is being identified as a hashtag or a mention
